Authentication

Login

To login, use this code:


  function (credentials) {
    return $http({
      method: "POST",
      url: "/loginreq",
      headers: {'Content-Type': 'application/x-www-form-urlencoded'},
      data: credentials
    })
    .then(function (res) {
        console.log(res);
      Session.create(res.data.id, res.data.user.id,
        res.data.user.role);
      console.log(res.data.user);
      return res.data.user;
    }, function(res) {
        return res.data;
      });
  };

Make sure that the object being passed in looks like this:

  {
    username: "usernamehere",
    password: "passwordhere"
  }

Dev Wars uses very simple login authentication and plans on changing that soon however until then, continue to use this endpoint to log in

You must replace the object with their login credentials

POST /loginreq

Query Parameters

Parameter Default Description
username NONE Username to sign in with
password NONE Password to sign in with

Reset Password


  return $http({
    method: "GET",
    url: "/resetpassword",
    data: {
      email: "theiremail"
    }
  });

Will send the user an email with a link to go to in order to actually change their password. This endpoint just starts the process

The above command returns a message structured like this:

  "Sent email"

You don’t have to be signed in to do this obviously

GET /resetpassword

Query Parameters

Parameter Default Description
email NONE The email of the user that would like to reset his/her password

Change Password


  return $http({
    method: "POST",
    url: "/changepassword",
    data: {
      newpassword: "thenewpassword"
    }
  });

Will change the user’s password in the database.

The above command returns a message structured like this:

  "Congratulations username, you have reset your password!"

You must have the cookie that is given when the user clicks the link in the email

POST /changepassword

Query Parameters

Parameter Default Description
newpassword NONE The new password for the user

Sign Up


  return $http({
    method: "POST",
    url: "/signupreq",
    data: {
      username: "username",
      password: "password",
      email: "email",
      recaptcha_response_field: "The recaptcha response"
    }
  });

Sends the user an email to verify their account. Sets their role to PENDING in the system.

The above command returns a message structured like this:

  "We have sent an email to theiremail"

Make sure you have implemented the new Google Recaptcha

POST /signupreq

Query Parameters

Parameter Default Description
username NONE The username for the user
password NONE The password for the user
email NONE The email for the user
recaptcha_response_field NONE The response from the recaptcha

User

User Role


  return $http({
    method: "GET",
    url: "/userrole",
  });

Returns the role for the user that is currently signed in.

The above command returns a message structured like this:

  "ROLE"

The user must be signed in for this to work, doesn’t matter what role they are.

POST /userrole

Query Parameters

Games

    return $http({
      method: "GET",
      url: "/games",
    });

The above command returns JSON structured like this:

{  
   "1419745055000":{  
      "teams":{  
         "red":{  
            "embedLink":"someEmbedLinkHere",
            "endgame":null,
            "players":{  
               "css":"cssUser",
               "js":"jsUser",
               "html":"htmlUser"
            },
            "status":"Fetching"
         },
         "blue":{  
            "embedLink":"someEmbedLinkHere",
            "endgame":null,
            "players":{  
               "css":"cssUser",
               "js":"jsUser",
               "html":"htmlUser"
            },
            "status":"Fetching"
         }
      },
      "name":"Game Name",
      "active":true,
      "timestamp":1419745055000

This endpoint retrieves all games with a limit of 50.

HTTP Request

GET /games

Query Parameters

Parameter Default Description
order desc The order by date for the results to come back (asc or desc)
offset 0 The offset of the results. If you want “the next page”, just offset by the limit(50)

Activate Game

  return $http({
      method: "GET",
      url: "/activategame",
      data : {
        timestamp : "timestamphere",
        enabled : true
    }
    });

The above command returns a message structured like this:

  "Set the game active to true/false"

Will set the given game to active and set it to the current game

You must be signed in and be at least a Moderator to do this

It will set all other games to inactive

HTTP Request

GET /activategame

Query Parameters

Parameter Default Description
timestamp NONE The timestamp of the game that you would like to activate
enabled true Activate the game or deactivate the game

Queue

Open the Queue

  return $http({
    method: "GET",
    url: "/openqueue",
    data: {
      enabled : true
    }
  });

The above command returns a message structured like this:

  "Successfully opened / closed the queue"

Will open the queue for the match to begin allowing users to apply for the game

You must be signed in and be at least a Moderator to do this

Disabling the queue will delete all applications for the current game

HTTP Request

GET /openqueue

Query Parameters

Parameter Default Description
enabled NONE boolean to either open or close the queue

Ranking List

      order: "asc",

  });

The above command returns JSON structured like this:

Gets a list of the users and their points There is a request limit of 50, use the offset param to get next

HTTP Request

GET /rankinglist

Query Parameters

Parameter Default Description
order desc The order of points by which the results come back
offset 0 The number that the results will be offset due to the request limit

Add Points

  return $http({
    method: "GET",
    url: "/addpoints",
    data: {
      username: "username to add the points to",
      points: 5
    }
  });

The above command returns a message structured like this:

  "username now has somenumber points"

Either adds or removes points from the user (Remove by using a negative number)

You must be signed in and be at least a Moderator to do this

HTTP Request

GET /addpoints

Query Parameters

Parameter Default Description
username NONE The username that the points will be added to
points NONE The amount of points to add to the user (Use negative to remove)
